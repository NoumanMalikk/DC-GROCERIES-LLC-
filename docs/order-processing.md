# Order Processing

Fulfilment workflow from payment to delivery for DC Groceries.

## Order lifecycle

```
pending_payment → paid → processing → shipped → delivered
                      ↘ cancelled / refunded
```

Status types defined in `src/lib/orders.ts`.

## Order reference format

- Prefix: `DCG-`
- 12 alphanumeric characters (no ambiguous I/O)
- Generated by `generateOrderReference()`
- Validated by `isValidOrderReference()` and contact form schema

## 1. Payment received

**Trigger:** Stripe webhook `payment_intent.succeeded`

Actions:

1. Verify webhook signature (`constructWebhookEvent`)
2. Idempotency check — skip if order already `paid`
3. Update order status → `paid`
4. Send confirmation email via Resend — [email-setup.md](email-setup.md)
5. Notify fulfilment queue (email, dashboard, or Supabase if configured)

## 2. Order review

Before picking:

- [ ] All line items `productionReady`
- [ ] No refrigerated items unless cold-chain active — [refrigerated-product-readiness.md](refrigerated-product-readiness.md)
- [ ] Shipping address in serviceable zone
- [ ] Fraud/AVS check passed (Stripe Radar)

Hold order and contact customer if:

- Address incomplete
- Refrigerated item in cart but cold-chain unavailable
- Item out of stock

## 3. Pick and pack

| Item type | Handling |
|-----------|----------|
| Produce bags | Check weight sticker; substitute **disabled** (`substitutionsEnabled: false`) |
| Weight-based | Weigh at pick; adjust charge if live weight-based pricing enabled |
| Clamshells | Inspect for crush damage |
| Cans/pantry | Check dent/leak |
| Refrigerated | Pull from cold storage; pack with ice |

Record on order:

- Actual weights for weight-based lines
- Lot/expiration for refrigerated (internal log)

## 4. Shipping

1. Generate shipping label (carrier integration TBD)
2. Update order → `shipped`
3. Store `trackingNumber` and `trackingUrl` on order
4. Send shipping notification email
5. Customer tracks via [/track-order](/track-order) with reference + email

## 5. Delivery

- Update status → `delivered` on carrier confirmation (or manual)
- Archive order record per retention policy

## Demo mode orders

Orders created with `demoMode: true`:

- Prefix confirmation emails with demo notice
- Do not fulfil or ship
- Use for checkout QA only

## Cancellations and refunds

| Scenario | Action |
|----------|--------|
| Before pick | Cancel → `cancelled`; void/refund PaymentIntent |
| After ship (perishable) | Follow returns policy — [legal-review.md](legal-review.md) |
| Partial refund | Stripe partial refund; update line items |

## Customer contact

Contact form accepts optional `orderReference` (`DCG-…` format).

Subjects: general, order, product, allergen, shipping, returns.

## Data stored per order

```typescript
{
  reference, status, customer, shippingAddress,
  items: [{ productId, sku, title, quantity, unitPriceCents, lineTotalCents }],
  subtotalCents, shippingCents, taxCents, totalCents,
  stripePaymentIntentId, trackingNumber, trackingUrl,
  createdAt, updatedAt, demoMode
}
```
